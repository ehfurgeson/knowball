The Court as a Graph: Decoding NBA Context with GATv2-TCN

Author

Eli Houston Furgeson

Overview: Basketball as an Evolving Network

Basketball is not played in a spreadsheet, it is played on a court where every movement, pass, and defensive switch connects players in a dynamic web of influence. Predicting how a player will perform involves understanding a complicated system of interdependencies between teammates and rivals. A player’s output isn’t just a product of their own skill, but a result of who they are sharing the floor with.

This document outlines the steps of reproducing and expanding upon a novel deep learning method, the Graph Attention Network with Temporal Convolutions (GATv2-TCN), originally proposed by Rui Luo and Vikram Krishnamurthy in a paper submitted to arxiv on March 29th, 2023 titled “Who You Play Affects How You Play: Predicting Sports Performance Using Graph Attention Networks With Temporal Convolution” https://arxiv.org/abs/2303.16741.

While traditional parametric models view player statistics as isolated time series, this approach structures each game as a dynamic player interaction graph. By explicitly modeling these relationships, we can capture the invisible context of the game. Furthermore, by expanding the original paper’s dataset to include more recent seasons and introducing an adaptive learning rate scheduler, we test the true robustness of this architecture.

1. Defining the Court: Architecture & Methodology

The underlying model is a GATv2TCN that maps a 10-day lookback window of player interactions to a single-day forward prediction.

  • Graph Construction: We don’t just look at team rosters, instead we map actual interactions. Edges are drawn between any players (teammates or opponents) who logged at least 10 minutes in the same game.
  • Attention Mechanism: A graph attention network calculates the specific “attention” or influence players exert on one another, allowing the model to learn defensive matchups and offensive synergies.
  • Temporal Convolution: To handle the multivariate time series of player statistics over the season, a temporal convolution layer provides the model with temporal predictive power.
  • Robust Data Split: We use a 50/25/25 chronological split, partitioned strictly by season boundaries to prevent data leakage and prove the model’s robustness across entirely different NBA seasons.

4. Translating the Game into Topology

Here, we convert our game logs into mathematical structures. The NetworkX graphs are transformed into edge-index tensors that PyTorch Geometric can process.

Code 
# ── Temporal embeddings to device ─────────────────────────────
team_temporal_t = torch.FloatTensor(team_temporal).to(DEVICE)
pos_temporal_t  = torch.FloatTensor(pos_temporal).to(DEVICE)
n_pos = 3

# ── Convert NetworkX graphs to edge-index tensors ─────────────
def graphs_to_edges(G_seq, player_ids):
    nd = {pid: i for i, pid in enumerate(player_ids)}
    out = []
    for G in G_seq:
        edges = list(G.edges())
        if not edges:
            n = len(player_ids)
            out.append(torch.stack([torch.arange(n), torch.arange(n)]).long().to(DEVICE))
        else:
            s, d = zip(*edges)
            s = [nd.get(x, 0) for x in s]; d = [nd.get(x, 0) for x in d]
            out.append(torch.stack([
                torch.LongTensor(s + d), torch.LongTensor(d + s)]).to(DEVICE))
    return out

G_edges = graphs_to_edges(G_seq, player_ids)
print(f'Built {len(G_edges)} edge tensors.')

# ── Per-season Z-score normalization ──────────────────────────
X_seq_raw_copy = X_seq.copy()
X_seq = (X_seq - mu_per_day) / sd_per_day

5. Chronological Windowing for True Robustness

To ensure our model is truly learning the underlying dynamics of the game rather than just memorizing a specific era, we enforce strict season-isolated sliding windows. The model trains on the past, validates on the near future, and is tested on seasons it has never seen.

Code 
# ── Season boundaries ─────────────────────────────────────────
unique_seasons = []
season_ranges = {}
prev_season = None
for d, s in enumerate(day_seasons):
    if s != prev_season:
        if prev_season is not None:
            season_ranges[prev_season] = (season_ranges[prev_season][0], d)
        season_ranges[s] = (d, None)
        unique_seasons.append(s)
        prev_season = s
season_ranges[prev_season] = (season_ranges[prev_season][0], len(day_seasons))

for s, (lo, hi) in season_ranges.items():
    print(f'  {s}: days [{lo}, {hi})  ({hi - lo} days)')

# ── Active-player mask ────────────────────────────────────────
active_all = (X_raw_seq != 0).any(axis=-1)

# ── Season-isolated sliding windows ──────────────────────────
X_in_parts, X_out_parts = [], []
G_in_parts, G_out_parts = [], []
mask_parts = []
target_day_indices = []

for s in unique_seasons:
    lo, hi = season_ranges[s]
    n_days = hi - lo
    n_win = n_days - SEQ_LENGTH - OFFSET + 1
    if n_win <= 0:
        continue
    X_s = X_seq[lo:hi]
    Xi = sliding_window_view(X_s[:-OFFSET], SEQ_LENGTH, axis=0)
    Xo = X_s[SEQ_LENGTH + OFFSET - 1:]
    Gi = [G_edges[lo + t:lo + t + SEQ_LENGTH] for t in range(n_win)]
    Go = G_edges[lo + SEQ_LENGTH + OFFSET - 1:lo + SEQ_LENGTH + OFFSET - 1 + n_win]
    mask_s = active_all[lo + SEQ_LENGTH + OFFSET - 1:lo + SEQ_LENGTH + OFFSET - 1 + n_win]
    days_s = list(range(lo + SEQ_LENGTH + OFFSET - 1, lo + SEQ_LENGTH + OFFSET - 1 + n_win))
    X_in_parts.append(Xi); X_out_parts.append(Xo)
    G_in_parts.extend(Gi); G_out_parts.extend(Go)
    mask_parts.append(mask_s)
    target_day_indices.extend(days_s)

X_in  = torch.FloatTensor(np.concatenate(X_in_parts, axis=0))
X_out = torch.FloatTensor(np.concatenate(X_out_parts, axis=0))
G_in  = G_in_parts
mask_out_t = torch.BoolTensor(np.concatenate(mask_parts, axis=0))
target_day_indices = np.array(target_day_indices)

# ── 50/25/25 chronological split ─────────────────────────────
T = len(G_in)
t1 = int(T * 0.50)
t2 = int(T * 0.75)

X_tr, y_tr, G_tr = X_in[:t1], X_out[:t1], G_in[:t1]
X_va, y_va, G_va = X_in[t1:t2], X_out[t1:t2], G_in[t1:t2]
X_te, y_te, G_te = X_in[t2:],  X_out[t2:],  G_in[t2:]
mask_tr = mask_out_t[:t1]
mask_va = mask_out_t[t1:t2]
mask_te = mask_out_t[t2:]

6. Instantiating the Network

Code 
from gatv2tcn import GATv2TCN

model_in = len(FEATURE_COLS) + 2 + 2  # 17 features total
n_pos = 3

team_emb = nn.Linear(n_teams, 2).to(DEVICE)
pos_emb  = nn.Linear(n_pos, 2).to(DEVICE)
model    = GATv2TCN(
    in_channels=model_in, out_channels=6,
    len_input=SEQ_LENGTH, len_output=1,
    temporal_filter=64, out_gatv2conv=32,
    dropout_tcn=0.25, dropout_gatv2conv=0.5, head_gatv2conv=4,
).to(DEVICE)

# Load Model Weights

7. Evaluating Graph Context

By testing the model against raw box score predictions, we can evaluate if the graph attention mechanism successfully deciphered the context of the matchups. Because we only evaluate active players (to prevent zeros from artificially inflating our accuracy), these metrics represent a rigorous test of the network’s predictive power.

Code 
all_preds_raw, all_trues_raw = [], []
all_preds, all_trues = [], []

with torch.no_grad():
    for i in range(len(X_te)):
        day_i = target_days_te[i]
        input_days = list(range(day_i - SEQ_LENGTH, day_i))
        Xl = []
        for t_step, abs_day in enumerate(input_days):
            tv = team_emb(team_temporal_t[abs_day])
            pv = pos_emb(pos_temporal_t[abs_day])
            Xl.append(torch.cat([X_te[i, :, :, t_step].to(DEVICE), tv, pv], 1))
        x = torch.stack(Xl, -1)[None, ...]
        p_latent = model(x, G_te[i])[0].cpu().numpy()
        t_latent = y_te[i, :, PRED_INDICES].numpy()

        mu_d = mu_per_day[day_i, 0, PRED_INDICES]
        sd_d = sd_per_day[day_i, 0, PRED_INDICES]
        p_raw = (p_latent * sd_d) + mu_d
        t_raw = (t_latent * sd_d) + mu_d

        m = mask_te[i].numpy()
        all_preds_raw.append(p_raw[m])
        all_trues_raw.append(t_raw[m])
        all_preds.append(p_latent[m])
        all_trues.append(t_latent[m])

AP_raw = np.concatenate(all_preds_raw)
AT_raw = np.concatenate(all_trues_raw)
AP = np.concatenate(all_preds)
AT = np.concatenate(all_trues)

# ── Raw-space metrics ─────────────────────────────────────────
rmse_raw = root_mean_squared_error(AT_raw, AP_raw)
mae_raw  = mean_absolute_error(AT_raw, AP_raw)
mape_raw = mean_absolute_percentage_error(AT_raw, AP_raw)

corr_z = []
for mi in range(6):
    r = np.corrcoef(AP_raw[:, mi], AT_raw[:, mi])[0, 1]
    if not np.isnan(r) and abs(r) < 1 - 1e-7:
        corr_z.append(np.arctanh(r))
corr_raw = np.tanh(np.mean(corr_z)) if corr_z else float('nan')

repro_metrics_raw = {
    'RMSE': float(rmse_raw), 'MAE': float(mae_raw),
    'MAPE': float(mape_raw), 'CORR': float(corr_raw),
}

# ── Raw-space metrics for comparison table ────────────────────

for k, v in repro_metrics_raw.items():

9. Conclusion: The Future of Sports Analytics is Connected

The success of the GATv2-TCN architecture proves that the relationships between players are just as highly predictive as the players themselves. By treating the NBA court not as a flat surface of isolated events, but as a rich, temporal graph of interactions, we capture the nuances of the game that traditional models miss from the gravity of a superstar drawing double teams to the defensive anchor shifting the geometry of the floor.

Adding more recent seasons to the dataset did not confuse the model, instead it strengthened it. This robustness highlights the incredible potential of graph network analysis in sports data science. When we embrace the interconnected nature of the game, we are better able to model the way that the game is actually played.
